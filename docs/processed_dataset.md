# Processed Data

This document describes the processed data for the dataset collected in [*Learning a Particle Dynamics Model with Real-world Videos*](https://arxiv.org/abs/2605.23845) (CVPR 2026 Findings), prepared for the released code repository [chkim403/gs-particle-dynamics](https://github.com/chkim403/gs-particle-dynamics).

Scene counts:

- `bowling/`: 297 scenes
- `cube_stacks/`: 232 scenes
- Total: 529 scenes. For our experiments, we used 292 bowling scenes and 210 cube-stacks scenes; the remaining scenes were excluded because of issues observed in their preprocessed outputs, such as segmentation failures. The list of excluded scenes is available as `skip_list` in the released code (`tools/dataset.py`).


`<split>` is either `train` or `test`.

## Scene-Level Files and Folders

`camera_meta.json` contains camera metadata shared by the scene:

```json
{
    "h": 480,
    "w": 640,
    "cam_id": ["234322305266", "248622303451", "336222300744", "336522303601"],
    "k": [...],
    "w2c": [...]
}
```

`rgb/` and `seg/` are organized by camera ID:

```text
rgb/<camera_id>/*.png
seg/<camera_id>/*.png
```

For each scene/camera, `rgb/` and `seg/` contain the selected interaction frame range. The `rgb/` images are masked RGB images that keep only the foreground objects, using the segmentation masks in `seg/`. Original unmasked images are available in the separately released raw dataset. These folders contain three more frames than the corresponding `gs/` folder, with the extra frames appended at the end of the sequence. We do not store Gaussians for these final frames because the input Gaussian trajectories used by our model are generated by applying the transformations stored in `obj_poses/` to static Gaussians.

`obj_poses/` contains per-camera object pose data:

```text
obj_poses/<camera_id>/obj_poses.npz
```

## GS Frame Folders

Each `gs/<frame>/` folder contains two files:

- `params_coarse.npz`: Gaussian parameters for that frame.
- `gs_soft_ids_coarse.npz`: object-ID assignments for the Gaussians.

Frame folder names are numeric video frame indices, for example:

```text
gs/35/params_coarse.npz
gs/35/gs_soft_ids_coarse.npz
```
